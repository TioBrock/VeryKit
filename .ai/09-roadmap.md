# Roadmap

## Roadmap Philosophy

VeryKit should grow by adding tools that solve frequent, concrete problems for QA Engineers, developers, automation engineers, and students.

The roadmap should prioritize:

- Daily usefulness.
- Low-friction workflows.
- Local-first processing.
- Strong search demand.
- Easy contribution.
- Consistent UX.

## Prioritization Criteria

Prioritize a tool when:

- It solves a frequent task.
- It can run client-side.
- It is easy to explain.
- It has strong search intent.
- It benefits QA/development workflows.
- It can share existing UI patterns.
- It does not require accounts or persistence.

Deprioritize a tool when:

- It requires sensitive remote processing.
- It depends on unstable external APIs.
- It is mostly content with little utility.
- It requires heavy dependencies.
- It breaks the calm product focus.

## Roadmap Review Cadence

The roadmap should be reviewed periodically as:

- User requests arrive.
- Contributors propose tools.
- Search demand becomes clearer.
- Maintenance load changes.
- Architecture matures.

Roadmap changes should keep the product philosophy intact.
